# 对话消息分层记忆

对话消息按三层记忆管理：短期记忆、中期记忆、长期记忆。

```html
graph LR
    A[用户消息] --> B{会话ID}
    B --> C[短期记忆<br/>（当前任务状态）]
    B --> D[中期记忆<br/>（工具会话/文件句柄）]
    B --> E[长期记忆<br/>（用户偏好/历史摘要）]
```

# 信息存储策略

## 存储策略
- 短期记忆保存在进程内存，按 `session_id` 维护最近上下文，不做磁盘持久化。
- 中期记忆持久化，并支持 `zvec` 检索；每次对话前必须按当前问题检索相关中期记忆并注入上下文。
- 长期记忆持久化，并支持 `zvec` 检索；在关键步骤（如规划、重规划）或失败场景下，检索并注入相关长期记忆。
- 当 `zvec` 不可用时，允许回退到本地关键词检索，不得中断主对话流程。

## 持久化目录
- 记忆数据按朋友 id 隔离，统一落在 `data/companions/<companion_id>/memory`。
- 建议最小目录结构：
  - `data/companions/<companion_id>/memory/mid_term.jsonl`
  - `data/companions/<companion_id>/memory/long_term.jsonl`
  - `data/companions/<companion_id>/memory/mid_term_zvec/`
  - `data/companions/<companion_id>/memory/long_term_zvec/`

## 写入与读取时机
- 对话开始前：
  - 读取短期记忆（当前会话）；
  - 检索中期记忆并注入上下文。
- 执行关键步骤前：
  - 检索长期记忆并注入上下文，作为策略选择与失败规避参考。
- 对话结束后：
  - 将本轮关键执行轨迹写入中期记忆；
  - 将稳定偏好、可复用经验摘要写入长期记忆。
- 执行失败后：
  - 必须补写失败摘要到长期记忆，供后续失败规避检索使用。
