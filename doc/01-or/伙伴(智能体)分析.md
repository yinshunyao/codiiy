# 伙伴系统需求分析（伙伴 ≠ 智能体）

## 1. 定位与边界
- 伙伴是“可配置的角色主体”，接近用户侧角色，不等同于某个智能体目录。
- 智能体（`agents/*`）与工具组件（`component/*`、`tools/*`）是伙伴可调用的能力来源，不是伙伴本体。
- 伙伴应具备独立的人设、行为约束、记忆摘要、知识库路径和能力白名单。

## 2. 目标
1. 在聊天一级菜单下提供“伙伴”二级管理入口，支持伙伴列表管理。
2. 支持新增伙伴，并配置角色属性、可调用能力、知识库关联信息。
3. 伙伴应支持记忆字段与知识库路径配置，并可对该知识库执行整理分析。
4. 保持与现有 `agents`、`component`、`tools` 能力体系兼容，避免重复建设。

## 3. 核心需求
### 3.1 伙伴实体
- 伙伴需持久化存储，至少包含：
  - 标识与展示：`name`、`display_name`
  - 角色信息：`role_title`、`persona`、`tone`
  - 记忆信息：`memory_notes`
  - 能力配置：可调用智能体模块、可调用工具组件模块
  - 知识配置：`knowledge_path`（指向项目内知识目录）
  - 生命周期：启用状态、创建时间、更新时间

### 3.2 能力关系
- 伙伴配置“可调用智能体模块”（如 `mindforge`、`helm`）。
- 伙伴配置“可调用工具组件模块”（如 `communicate`、`observe`、`decide`、`handle`）。
- 默认按白名单生效，未配置模块不可调用。

### 3.3 知识库与记忆
- 伙伴知识库目录由 `knowledge_path` 指定，路径应限制在当前项目根目录内。
- 伙伴支持记录“长期记忆摘要/行为备忘”，用于后续提示词拼装。
- 页面支持对伙伴知识库执行整理并回显报告摘要。

### 3.4 管理能力
- 伙伴列表：展示角色配置、能力白名单、知识库关联状态。
- 新增伙伴：在页面创建并保存配置。
- 编辑伙伴：修改角色信息、能力白名单、知识路径、记忆信息。

### 3.5 列表展示密度优化
- 伙伴列表卡片改为“紧凑信息布局”，减少逐行字段堆叠导致的卡片高度过高问题。
- 核心信息按优先级分层展示：第一层展示名称/启用状态，第二层展示角色、模型与知识库状态，第三层展示能力摘要与更新时间。
- 保留编辑入口，避免因布局压缩影响关键操作可达性。

### 3.6 伙伴专属聊天窗口
- 在“伙伴列表”中每个伙伴的“编辑配置”左侧提供 `聊一聊` 操作按钮，点击后以新窗口弹开。
- 在“编辑伙伴”页面保留 `聊一聊` 操作按钮，点击后以新窗口打开。
- 新窗口聊天页复用主聊天窗口交互样式（消息区、输入区、异步回复、语音与附件能力保持一致）。
- 伙伴聊天会话独立于主聊天会话：按“当前登录用户 + 当前伙伴”维度隔离，消息不与主聊天混写。
- 仅允许在“主聊天会话 / 伙伴会话”中发送消息；总结会话保持只读。
- 伙伴聊天页应明确展示当前伙伴标识，避免用户误将消息发送到主聊天。
- 伙伴聊天页标题不展示“伙伴聊天”等通用前缀，直接展示伙伴名称。
- 伙伴聊天页中助手侧消息角色名不使用“助手”通用文案，改为对应伙伴名称。
- 伙伴聊天执行模式从“单轮对话”升级为“自主规划 + 多智能体协同执行 + 汇总回复”。
- 伙伴聊天中的“心法/号令/技能/工具组件”必须进入真实执行链路，不能只作为提示词文案展示。

### 3.7 聊天菜单与朋友通讯录视图
- 左侧导航将“朋友聊天记录”统一归入“聊天”一级菜单，不再单独提供“朋友”一级菜单。
- “聊天”一级菜单下固定置顶“主聊天会话”（大群语义），保持现有入口与行为不变。
- 主聊天会话下方按二级列表展示伙伴聊天记录，样式参考常见通讯录：头像、名称、消息摘要、最近消息时间。
- 二级伙伴聊天条目点击后进入对应伙伴聊天会话。
- 伙伴通讯录展示顺序按“最近聊天消息时间”倒序；无聊天记录的伙伴排在已聊天伙伴之后。
- 聊天窗口中的伙伴列表视图使用“头像 + 名称 + 消息摘要 + 时间”紧凑列表样式，避免大卡片信息堆叠。

## 4. 非功能要求
- 伙伴配置应通过 ORM 持久化。
- 伙伴知识目录必须做路径安全校验，禁止越界访问项目目录外路径。
- 设计与实现优先复用现有页面框架、知识库整理工具和侧栏导航能力。
- 伙伴聊天在执行复杂任务时应支持步骤化执行与失败降级，保证可回复性与可观测性。

## 5. 交付范围（本阶段）
- 聊天 > 伙伴 二级菜单入口
- 伙伴列表页
- 新增伙伴页
- 编辑伙伴页
- 伙伴知识库整理入口与结果展示
- 伙伴聊天自主规划器（Custom Planner）
- 伙伴聊天协同调度器（Coordinator）
- 心法与号令执行适配器（mindforge/helm）
- 工具组件执行器（component/tools 白名单调用）
