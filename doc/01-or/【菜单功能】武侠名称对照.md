# 背景

- 本平台主要定位为个人助手，智能体全部拟人化，取大侠名字
- 用户希望整个平台的术语和菜单名称能够在"标准中文"与"武侠风"之间自由切换，增强沉浸感和趣味性
- 未来可能扩展更多命名风格（如科幻风、古典风等），需要一套可扩展的名称映射机制

# 目标

1. 支持平台术语的多套名称配置，系统预置"标准中文"和"武侠风"两套方案，默认使用武侠风
2. 支持用户在个人设置中切换名称方案，并支持用户自建新的名称方案
3. 调整一二级菜单名称，使之与所选名称方案联动显示

# 要求

## 术语映射表

平台核心术语的英文、标准中文和武侠风名称对照如下：

| 英文术语     | 标准中文         | 武侠名称       | 说明 / 隐喻 |
|--------------|------------------|------------------|-------------|
| codiiy       | codiiy平台       | 开智枢           | 承接 `openclaw`，"爪"为标识，"江湖"为舞台 |
| Agent        | 智能体           | 大侠             | 每个 Agent 都是独当一方的大侠（如"Cursor 大侠"） |
| Adapter      | 适配器           | 通义使           | 负责沟通各大侠的"外交使者" |
| Skill        | 技能             | 绝学             | 每个大侠掌握的独门功法（如"降龙十八掌"） |
| Tool         | 工具             | 兵刃             | 代码编辑器是"青钢剑"，搜索是"千里镜" |
| Component    | 组件             | 机枢             | "机"指机关/机制，"枢"为关键部件 |

## 组件子分类映射

| 标准中文     | 武侠名称   | 说明 |
|------------|----------|------|
| 沟通       | 传音     | "隔空传音"，指与外部系统通信 |
| 观察       | 望气     | "望气术"——看数据、看状态 |
| 决策       | 运筹     | "运筹帷幄"，决策中枢 |
| 操作       | 出招     | "一招制敌"，执行动作 |

## 武侠风菜单结构

括号内为标准中文名称，界面不显示，鼠标悬停时以 tooltip 形式提示：

- 江湖（聊天）
  - 华山论剑（广场）
  - 大侠A（最近的好友会话）
  - 大侠B（最近的好友会话）
- 武库
  - 大侠
  - 通义使
  - 绝学
  - 兵刃
- 机枢
  - 传音
  - 望气
  - 运筹
  - 出招

武库菜单示例：UI 显示"武库"，Hover 提示"大侠、兵刃、绝学、通义使——尽在此库"

## 功能性要求

1. 名称方案以 JSON 文件存储在 `codiiy/core/settings` 目录下
2. 预置方案不可删除，用户自建方案可增删改
3. 切换名称方案后，菜单和界面术语需实时联动刷新，无需重启
4. 未在映射表中定义的菜单项，武侠风方案可后续补充，暂时显示标准中文名称
