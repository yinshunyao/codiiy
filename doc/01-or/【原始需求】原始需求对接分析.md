# 目标
1. 实现一个会话 web 程序，用于原始需求收集和整理，需要持久化存储
2. 会话处理
- 对话的原始记录存储到数据库中，界面会展示会话内容
- 根据会话的每次内容，整理用户的需求
- 每次对话需要记录上下文，用户的每次对话，需要按照如下逻辑处理
  - 如果理解了没有疑问就回复 "已收到，你描述完了么？"
  - 如果有疑问或者问题，提出问题，弄清楚用户表达内容
- 需要根据需求内容制定合适的名字，放到目录原始目录下
3. 数据库先支持 SQLite
4. 默认 admin  123456，支持用户登录和密码修改
5. 项目管理功能
- 支持多个项目管理，默认当前项目为 core
- 可以选择其他目录作为新的项目处理，项目路径默认与 core 保持同级
- 每个对话归属于特定的项目
- 支持项目的新增、编辑、删除、切换和设置默认项目
- 新建项目时，关键目录（doc、tools、data/roles）需要同步到新的项目工程中
- 增加会话删除功能
- 会话需要支持多模态，前端界面和后端都需要按照主流的多模态对话支持

# 要求
1. 充分利用 Django 框架的特性，不要重复开发
2. 资源管理器的管理查询结果必须以当前项目目录下的实际文件与目录为准，数据库记录仅可作为补充信息，不可单独作为列表数据来源
3. 资源管理器需支持按照文件夹层级浏览（下钻、返回上级），并且访问范围必须锁定在当前项目目录内，禁止越界访问
4. 资源管理器下方预览区需提供“编辑”能力：对文本类文件（如 `.md`、代码文件等）允许在线修改并保存；非文本文件保持只读预览提示
