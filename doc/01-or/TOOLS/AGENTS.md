# 目录用途
本目录用于沉淀 `TOOLS` 相关的原始需求文档，作为工具能力建设与治理的需求源头，服务于需求阶段（01-or）。

# 文档范围
允许放入：
1. 工具新增能力需求。
2. 工具调用规范、参数规范、测试规范等原始需求。
3. 与工具治理相关的需求拆分文档（按“一需求一文档”原则）。

不应放入：
1. 具体实现设计与技术方案细节（应放到 `codiiy/doc/02-dr/`）。
2. 测试用例与测试步骤细节（应放到 `codiiy/doc/03-tr/`）。
3. 发布记录与运维操作说明（应放到 `codiiy/doc/99-ops/`）。

# 编辑规则
1. 文档命名建议使用 `工具-<主题>.md` 或 `【工具】<主题>.md`，避免笼统命名。
2. 每个 OR 文档必须保持固定章节顺序：
   - `# 背景`
   - `# 目标`
   - `# 要求`
   - `# 当前工作项`
   - `# 已完成工作项`
3. 新需求优先判断是否应新建文档；若为已有需求补充，需融入原章节，不得在文末堆叠“变更日志”。
4. 任务完成后必须将条目从 `# 当前工作项` 迁移到 `# 已完成工作项`。

# 协作约定
1. `01-or/TOOLS` 仅定义“要做什么”，不定义“怎么做”。
2. 本目录需求一旦新增或调整，必须同步评估并更新：
   - `codiiy/doc/02-dr/` 对应开发设计文档；
   - `codiiy/doc/03-tr/` 对应测试设计文档。
3. 若需求变更影响部署、启动参数或运行约束，需同步检查 `codiiy/doc/99-ops/`。
