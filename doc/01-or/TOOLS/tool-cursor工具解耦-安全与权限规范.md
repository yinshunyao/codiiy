# 背景
Cursor 工具测试与调用会触发真实能力执行。若缺少统一安全约束，容易产生敏感信息泄露、越权调用、错误处理不清晰等风险，影响平台治理与使用安全。

# 目标
1. 明确 Cursor 工具的安全边界和权限拦截要求。
2. 统一敏感信息处理与异常反馈口径。
3. 确保工具能力在可用性的同时满足最小安全约束。

# 要求
## 功能性要求
1. 当工具声明系统权限且权限不满足时，调用必须在执行前被拦截，并返回明确缺失权限提示。
2. 工具结果展示必须支持敏感字段脱敏，至少覆盖 `password`、`secret`、`token`、`api_key` 等字段。
3. 调用异常需要区分“权限失败、参数失败、执行失败”三类结果，避免统一为模糊错误。
4. 安全策略调整时，必须同步更新相应设计文档和测试设计文档中的安全测试范围。

## 非功能性要求
1. 安全策略不应引入绕过现有调用链的临时入口。
2. 安全拦截与脱敏规则应具备可扩展性，支持后续新增敏感字段和权限类型。
3. 安全相关需求必须可被自动化验证，避免仅依赖人工检查。
