# v0.1.0 20260225

## 新增功能
1. 新增基于 Django 的原始需求收集会话 Web 系统。
2. 新增会话能力：会话列表查看、会话新建、SQLite 持久化存储。
3. 新增账号能力：用户登录、退出登录、密码修改。
4. 新增默认管理员初始化：首次迁移后自动创建 `admin / 123456`（用户表为空时）。
5. 新增 Django Admin 对会话数据的管理支持。

## 变更功能
1. 按 `core/rules.md` 规范补齐文档链路：`01-or`、`02-dr`、`03-tr`、`99-ops`。
2. 统一以 Django 内置认证与模板机制实现，不重复开发认证流程。

## 解决问题
1. 解决“原始需求缺少统一会话收集入口”的问题。
2. 解决“需求收集数据无法持久化存储”的问题。
3. 解决“缺少基础账号登录与密码维护能力”的问题。

## 遗留问题
1. 当前版本仅支持 SQLite，尚未支持 MySQL/PostgreSQL。
2. 目前为基础 MVP 页面，未引入复杂权限分级与流程审批能力。
3. 本地运行依赖需手动安装（见 `doc/99-ops/run.md`）。
