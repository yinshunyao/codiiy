# 运维与发布 (Operations, OPS) 规范

**文件路径**: `doc/99-ops/AGENTS.md`
**目的**: 定义运维、部署、发布相关文档的管理标准。
**状态**: 已采纳 (Adopted)

## 目录用途

本目录存放所有与**运维部署、环境配置、版本发布**相关的文档和配置文件。用于记录系统运行所需的环境信息、部署步骤、发布版本说明等运维知识。

## 文档范围

### 允许放入
- 版本发布说明（Release Note）
- 部署配置文件和说明（Nginx 配置、Django 启动命令等）
- 环境依赖说明（requirements.txt、数据库初始化脚本等）
- 生产环境信息和运维操作指南
- 对外接口对接文档

### 不应放入
- 原始需求（属于 01-or）
- 开发设计方案（属于 02-dr）
- 测试设计和用例（属于 03-tr）
- Bug 分析记录（属于 91-qa）

## 编辑规则

### 文档分类

| 类型 | 示例 | 说明 |
| :--- | :--- | :--- |
| 版本发布 | `release Note.md` | 按版本号倒序记录，每个版本包含新增/变更/解决问题 |
| 部署指南 | `run_django.md`、`部署问题说明.md` | 记录启动命令、环境配置、常见问题 |
| 环境配置 | `ls.conf`、`requirments.txt`、`db.sql` | 配置文件和数据库脚本 |
| 对接文档 | `数据集导出链接对接文档.md` | 与外部系统的接口对接说明 |

### Release Note 格式约定
- 每个版本使用一级标题 `# vX.Y.Z YYYYMMDD`。
- 版本内容分为：`## 新增功能`、`## 变更功能`、`## 解决问题` 三个二级标题。
- 新版本追加在文件顶部，旧版本自然下沉。

### 发布记录维护策略（草稿先行）
- `doc/99-ops/release-note.md` 作为正式发布文档，不要求每次细小改动后立即更新。
- 日常开发中的细小“新增功能/变更功能/解决问题”统一先写入 `doc/99-ops/release-note-draft.md`。
- 当收到“发布整理”或“发布汇总”指令时，再从草稿中提炼、去重、精简后写入 `release-note.md`。
- 草稿文档应按时间顺序持续追加，避免遗漏；正式发布后可对已纳入版本的草稿项做归档标记。
