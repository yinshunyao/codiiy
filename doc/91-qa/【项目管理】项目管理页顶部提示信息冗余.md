# 问题描述
- 页面：`设置 > 项目管理`
- 现象：页面顶部出现多条提示信息（绿色消息列表），占用主要可视区域并干扰项目管理操作。
- 复现条件：执行会产生 `messages.success/messages.error` 的操作后进入项目管理页（如“设为默认项目”），顶部即渲染提示区。

|:---|:---|
| 原始需求 | `doc/01-or/【原始需求】原始需求对接分析.md` |

| 测试设计 | `doc/03-tr/01 会话原始需求收集系统测试需求.md` |

# 根因分析
- 模板 `core/templates/collector/project_list.html` 在主内容顶部包含 `{% if messages %}` 渲染块，会把 Django 消息框架中的临时消息逐条展示到页面最上方。
- 该提示区不属于项目管理核心信息，且在连续操作后会堆叠，导致页面可读性下降。

# 涉及文件
| 文件路径 | 修改摘要 |
|:---|:---|
| `core/templates/collector/project_list.html` | 删除顶部消息提示渲染块（`{% if messages %}`） |
| `test/test_core/test_collector/test_views.py` | 新增回归测试：触发“设为默认项目”后，项目管理页不再展示顶部提示文案 |
| `doc/02-dr/01 会话原始需求收集系统设计.md` | 补充项目管理页顶部提示区隐藏设计约束 |
| `doc/03-tr/01 会话原始需求收集系统测试需求.md` | 补充对应页面回归测试项 |

# 设计约束更新
- 项目管理页顶部不展示 Django flash message 文本列表；操作反馈不再占用页面主视图区域。
