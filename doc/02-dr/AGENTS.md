# 开发设计文档 (Design Review, DR) 规范

**文件路径**: `doc/02-dr/AGENTS.md`
**目的**: 定义开发设计文档的管理标准，确保设计方案与原始需求可追溯、与测试设计同步。
**状态**: 已采纳 (Adopted)

## 目录用途

本目录存放所有功能的**开发设计方案**，是原始需求（01-or）到代码实现的桥梁。每个功能模块的技术方案、接口设计、数据模型、业务流程等设计内容统一在此管理。

## 文档范围

### 允许放入
- 功能设计方案（含前端交互设计、后端接口设计、数据模型设计）
- 业务流程设计（状态流转、角色权限矩阵等）
- 技术选型说明和架构约束
- 设计相关的参考代码片段和范例

### 不应放入
- 原始需求描述（属于 01-or）
- 测试用例设计（属于 03-tr）
- Bug 分析与修复记录（属于 91-qa）
- 运维部署文档（属于 99-ops）

## 编辑规则

### 目录组织
按功能模块建立子目录，子目录使用 `编号 模块名` 格式：

| 编号 | 模块 | 说明 |
| :--- | :--- | :--- |
| 00 | 基础功能 | 界面框架、权限、文件访问等通用基础能力 |
| 01 | 任务创建 | 任务创建向导四步流程及各步骤配置 |
| 02 | 数据资产 | 数据源管理、数据标准、业务场景 |
| 03 | 标注 | 标注任务分发、标注模式、文件列表 |
| 04 | 质检 | 质检任务分发、质检模式 |
| 05 | 审核 | 审核任务流程 |
| 06 | 预标注 | 预标注业务流程、小模型/大模型方案 |
| 07 | 一致性计算 | 标注一致性统计 |
| 08 | 模型管理 | 内置模型服务管理 |
| 09 | 系统标注模板管理 | 系统级标注模板维护 |
| 10 | 高质量数据集管理 | 数据集库设计 |
| 99 | 业务设计/参考文档 | 跨模块业务设计和外部参考资料 |
| 00 备份 | 备份 | 废弃/被替代的旧版设计，仅供历史参考 |

### 文档命名
- 格式：`编号.子编号 设计主题.md`（如 `01.02. 任务创建.md`）。
- 模块内多个子设计文档按编号递增排列。

### 关联文档章节（强制）
每个设计文档**必须**包含 `# 关联文档` 章节，用表格列出对应的原始需求(01-or)、关联设计(02-dr)、测试设计(03-tr)文档路径。至少包含一条原始需求记录。

### 内容结构建议
设计文档建议包含：关联文档（强制）、功能设计、数据模型（如涉及）、约束与边界、关联设计引用。

### 变更流程
- 需求变更时，先更新 01-or，再同步修改本目录中的设计文档。
- 修改设计文档时，必须同步检查 03-tr 中对应测试设计是否需要更新。
- 增量变更应融入对应章节，禁止在文档末尾以日期为标题堆叠。

## 协作约定

| 相邻目录 | 关系 |
| :--- | :--- |
| 01-or | 上游输入：设计文档的每个功能点必须可追溯到原始需求 |
| 03-tr | 对等关系：开发设计与测试设计同步产出，模块一一对应 |
| 91-qa | Bug 修复涉及设计变更时，需同步更新本目录文档 |
| 99-ops | 设计中的部署约束和运维要求应在 99-ops 中同步说明 |
