# 功能设计

## 整体架构

adapter 模块位于 `codiiy/adapter/`，独立于 Django 应用（collector），通过 CompanionProfile 模型的字段与伙伴体系关联。

```
codiiy/adapter/
├── __init__.py          # 包初始化，导出 registry
├── base.py              # 基类 BaseAdapter，定义统一接口
├── types.py             # 类型定义（执行结果、配置上下文等）
├── registry.py          # 全局注册表，按 adapter_type 查找实现
├── process_adapter.py   # process 类型：命令行执行
├── http_adapter.py      # http 类型：HTTP 接口调用
└── cursor_local/        # cursor_local 类型：Cursor CLI 本地调用
    ├── __init__.py
    ├── adapter.py       # 执行、续跑、指令注入
    └── parser.py        # stream-json 解析与错误识别
```

## 基类设计（BaseAdapter）

```python
class BaseAdapter:
    adapter_type: str        # 类型标识，如 "process"
    label: str               # 展示名称，如 "通用命令行"
    config_doc: str           # 配置说明文档

    def execute(self, context: AdapterExecutionContext) -> AdapterExecutionResult:
        """执行 adapter 任务"""
        raise NotImplementedError

    def test_environment(self, config: dict) -> AdapterEnvironmentTestResult:
        """检测运行环境是否满足条件"""
        raise NotImplementedError

    @classmethod
    def get_config_schema(cls) -> dict:
        """返回该 adapter 的配置字段说明"""
        return {}
```

## 类型定义（types.py）

### AdapterExecutionContext
```python
@dataclass
class AdapterExecutionContext:
    run_id: str                     # 本次执行 ID
    companion_name: str             # 伙伴标识
    config: dict                    # adapter 配置（来自 companion.adapter_config）
    prompt: str                     # 输入提示词
    cwd: str = ""                   # 工作目录
    env: dict = field(default_factory=dict)  # 额外环境变量
    timeout_sec: int = 0            # 超时秒数，0=不限
```

### AdapterExecutionResult
```python
@dataclass
class AdapterExecutionResult:
    exit_code: int | None = None
    signal: str | None = None
    timed_out: bool = False
    error_message: str | None = None
    stdout: str = ""
    stderr: str = ""
    result_json: dict | None = None
```

### AdapterEnvironmentTestResult
```python
@dataclass
class AdapterEnvironmentTestResult:
    adapter_type: str
    status: str          # "pass" | "warn" | "fail"
    checks: list[dict]   # 各检查项
    tested_at: str        # ISO 时间戳
```

## 注册表设计（registry.py）

全局维护 `adapter_type -> BaseAdapter 实例` 的映射，支持：
- `register(adapter)` — 注册 adapter 实例
- `get_adapter(adapter_type)` — 按类型获取，未找到返回 None
- `list_adapters()` — 列出所有已注册 adapter
- `get_adapter_choices()` — 返回 Django form choices 格式

内置 adapter 在模块导入时自动注册。

## process adapter

基于 `subprocess` 模块执行命令行程序。

配置字段：
| 字段 | 类型 | 必填 | 说明 |
|:---|:---|:---|:---|
| command | str | 是 | 要执行的命令 |
| args | list[str] | 否 | 命令参数 |
| cwd | str | 否 | 工作目录，默认系统根目录 |
| env | dict[str,str] | 否 | 额外环境变量 |
| timeout_sec | int | 否 | 超时秒数，默认 0（不限） |
| grace_sec | int | 否 | SIGTERM 宽限秒数，默认 15 |

## http adapter

基于 `requests` / `urllib` 发起 HTTP 请求调用远程 agent 服务。

配置字段：
| 字段 | 类型 | 必填 | 说明 |
|:---|:---|:---|:---|
| url | str | 是 | 请求 URL |
| method | str | 否 | HTTP 方法，默认 POST |
| headers | dict[str,str] | 否 | 自定义请求头 |
| timeout_sec | int | 否 | 请求超时秒数，默认 30 |
| payload_template | str | 否 | 请求体模板，支持 {prompt} 占位符 |

## cursor_local adapter

基于 Cursor Agent CLI（默认命令 `agent`）调用本地 Cursor 能力，并解析 `stream-json` 输出。

配置字段：
| 字段 | 类型 | 必填 | 说明 |
|:---|:---|:---|:---|
| command | str | 否 | Cursor CLI 命令，默认 `agent` |
| cwd | str | 否 | 工作目录，默认上下文 cwd |
| model | str | 否 | 模型，默认 `auto` |
| mode | str | 否 | `plan` / `ask` |
| extra_args | list[str] \| str | 否 | 额外参数 |
| instructions_file_path | str | 否 | 启动前读取并拼接到 prompt 前 |
| session_id | str | 否 | 上次会话 ID，用于 `--resume` |
| session_cwd | str | 否 | 上次会话 cwd，不一致时放弃恢复 |
| workspace_id | str | 否 | 上层工作空间标识 |
| env | dict[str,str] | 否 | 额外环境变量 |
| timeout_sec | int | 否 | 超时秒数 |
| hello_probe | bool | 否 | 环境检测时执行 hello 探针 |

关键行为：
1. 启动参数固定包含 `-p --output-format stream-json --workspace <cwd>`。
2. 若 `extra_args` 未包含 `--trust`、`--yolo`、`-f`，自动补 `--yolo`。
3. 支持解析 `stdout: {...}` 前缀行，累计提取 `session_id`、`usage`、`cost_usd`、`summary`、`error_message`。
4. 若使用 `--resume` 且命中 unknown session 错误，自动无 `--resume` 重试一次。
5. 输出在 `result_json` 中回传 `session_id`、`session_params(cwd/workspace_id)`、`usage`、`cost_usd` 与恢复信息。

## 数据模型变更

CompanionProfile 新增字段：

| 字段名 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| adapter_type | CharField(40) | blank=True, default="" | adapter 类型标识，空值表示不使用 adapter |
| adapter_config | JSONField | default=dict, blank=True | adapter 配置参数，JSON 格式 |

## 伙伴表单变更

CompanionProfileForm 新增：
- `adapter_type`：Select 下拉框，选项从 adapter 注册表动态获取，含空白选项"不使用 adapter"
- `adapter_config`：Textarea（JSON 文本域），用户手工填写 adapter 配置

表单模板在基础信息区域之后、模型配置区域之前插入 adapter 配置区块。

# 约束与边界

1. 当前已实现 `process`、`http`、`cursor_local` 三种类型，其他特化 adapter（如 claude_local）不在本次范围
2. adapter 执行（execute）功能本次仅搭建接口骨架，不在伙伴对话流程中调用
3. adapter_config 暂时不做字段级校验，仅做 JSON 格式校验
4. adapter 模块不依赖 Django ORM，保持纯 Python 实现
