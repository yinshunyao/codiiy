# mindforge/auto_strategy

Auto（自动路由）策略目录（可独立导入/导出）。

## 目录职责

- `api.py`：策略入口（`AutoMindforgeStrategy`）。
- `__init__.py`：公共导出。

## 执行约束

- 自动路由采用“两段式判定”：
  - 第一段：先调用大模型进行任务意图分析，输出结构化路由决策（任务类型、首选策略、是否必须执行动作、原因）；
  - 第二段：当模型判定缺失、格式异常或与工具边界冲突时，再使用规则兜底，确保可执行性与稳定性。
- 自动分析任务复杂度与工具依赖，路由到最合适策略。
- 支持分层兜底：复杂任务优先 `plan_execute`，失败后可回退到 `reflexion` 与 `react`。
- 无工具场景优先走 `cot`，避免无效工具规划。
- 对执行型任务（存在动作意图 + 可执行工具链）必须优先走可执行策略，不允许仅以 `cot` 建议文本判定完成。
- 对执行型任务，阶段成功判定需满足“有效工具调用闭环”（至少一次无报错工具调用）。
- 对执行型任务，阶段返回 `success=true` 后仍需做“结果达标判定”（基于大模型结合任务目标与执行产出判断）；若判定不达预期，需触发同阶段改道重试或后续策略回退，不能直接结束。
- 对执行型任务，单阶段失败后会触发“替代方案重试”，尽量在不同思路下继续尝试，避免单次失败即提前结束。
- 路由步骤需记录可解释信息：模型判定摘要、能力标签、最终采用策略与触发原因，便于排障与复盘。
- 返回结构与 mindforge 运行结果保持一致（`success/final_answer/steps/error`）。
