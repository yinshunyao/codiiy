# 结构设计规划目录 AGENTS 规范（全局）

当创建、调整、或维护“整体结构设计规划”的文档目录时，必须同时维护该目录下的 `AGENTS.md`。

## 适用范围
- 任何承载结构设计、方案设计、规划说明、测试规划、运维规划的目录。
- 包含但不限于 `doc/` 下的分层目录（如 `01-or`、`02-dr`、`03-tr`、`99-ops`）及后续新增同类目录。

## 强制要求
1. 目录内必须存在 `AGENTS.md`，且与目录内容保持同步。
2. 新增此类目录时，首个提交必须包含 `AGENTS.md`（不得后补）。
3. 修改目录职责、文档分类、命名规则后，必须先更新 `AGENTS.md` 再编辑其他文档。
4. 需求分析之后整合到01-or目录中的相关文档
5. 开发设计整合到02-dr目录中的相关文档
6. 测试需求和设计整合到03-tr目录中的相关文档
7. 运维和发布相关内容整合到99-ops目录中的相关文档

## AGENTS.md 最少内容
- 目录用途：该目录解决什么问题、服务哪个阶段。
- 文档范围：允许放入的文档类型与不应放入的内容。
- 编辑规则：文档命名、章节约束、变更流程、完成项归档方式。
- 协作约定：与相邻目录的边界、引用关系、优先级说明。

## 执行约束
- 若发现目录缺失 `AGENTS.md`，应先补齐再继续该目录下的内容编辑。
- 若 `AGENTS.md` 与实际结构不一致，应先修正文档规则再改实现内容。
