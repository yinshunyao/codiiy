# 工具管理规范

## 权限声明
可请示修改

## 1. 工具目录结构
1. 每个工具独立的目录，例如 `tools/api_client`
2. 工具目录下必须包含如下文件：
   - `__init__.py`：Python 包初始化文件
   - 主代码文件：根据工具功能命名，例如 `api_client.py`
   - `requirements.txt`：工具依赖文档
   - `README.md`：工具使用说明文档

## 2. 工具实现规范
1. 工具类命名：使用驼峰命名法，例如 `ApiClient`
2. 方法命名：使用小写字母和下划线，例如 `get_response()`
3. 参数命名：使用小写字母和下划线，例如 `api_key`
4. 返回值：统一返回字典格式，包含 `success` 和 `data` 字段，例如 `{"success": true, "data": {...}}`
5. 错误处理：使用 try-except 捕获异常，并返回错误信息，例如 `{"success": false, "error": "错误信息"}`

## 3. 工具使用规范
1. 工具调用：通过导入工具类，创建实例后调用方法
2. 配置管理：支持通过环境变量或配置文件管理配置项
3. 日志记录：工具内部需要记录关键操作的日志
4. 测试用例：为工具编写测试用例，确保功能正常

## 4. 工具依赖管理
1. 工具依赖必须在 `requirements.txt` 文件中声明
2. 依赖版本应该明确指定，避免版本冲突
3. 依赖安装命令：`pip install -r requirements.txt`

## 5. 工具发布规范
1. 工具发布前必须进行充分测试
2. 工具发布时需要更新 `README.md` 文件，说明工具功能和使用方法
3. 工具发布时需要更新 `requirements.txt` 文件，确保依赖正确
